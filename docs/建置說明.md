## 3. 資料模型(RTDB)

```
users/{uid}:        { name, email, photo, isAdmin, status(pending|approved), createdAt }
projects/{pid}:     { title, description, startDate, endDate, emoji, color,
                      ownerId, ownerName, memberCount, members/{uid}:{status,role,name}, createdAt }
trips/{pid}:        { title, budget, days/{dayId}: { date, spots/{spotId}:
                      { name, type, time, note, cost, lat, lng, placeId?, poiId?, avoidToll? } } }
expenses/{pid}/{id}: 費用紀錄
photos/{pid}/{dayId}/{photoId}: { kind(planning|memory), url, storagePath, uploadedBy, uploaderName, uploadedAt }
pois/{poiId}:       { name, nameAliases?, placeId?, lat, lng, address, category, article,
                      photos/{id}:{url,storagePath,uploadedBy,uploadedAt}, links[],
                      createdBy, updatedBy, updatedAt, usedByCount }
poisHistory/{poiId}/{id}: { article, by, ts }   // 文章近 10 版
docs/{pid}/{docId}: { spotPoiId?, fileName, url, storagePath, size, mime,
                      uploadedBy, uploaderName, uploadedAt }   // 私人文件
```

- **景點順序** = spots key 字串排序(重排時 rekey 為 `spot_00000000` 補零;新增用 `spot_{timestamp}` 落尾)
- **標題單一來源**:trip.html 改標題會同步 `projects/{pid}/title`

## 4. Storage 路徑

```
pois/{poiId}/{photoId}.jpg              共用景點照片(登入即可讀)
projects/{pid}/days/{dayId}/{id}.jpg    每日照片
projects/{pid}/docs/{docId}_{fileName}  私人文件(≤10MB)
```

照片上傳前 client 壓縮(長邊 ≤1600px、JPEG 0.8)。

## 5. 安全規則

- **RTDB**(`database.rules.json`):根層全拒;`users` 防自我提權/自改 approved;`projects` approved 可讀、成員可寫;`trips`/`expenses`/`photos`/`docs` per-project 成員制;`pois`/`poisHistory` approved 可讀寫、刪除限管理員。
- **Storage**(`storage.rules`):全路徑需登入 + 大小/型別限制。
- ⚠️ **已知限制**:Storage rules 查不到 RTDB 成員名單,私人文件的成員限定靠「RTDB metadata 成員制 + 下載 URL 不外流」兩層;要做到 Storage 層 per-project 硬隔離需 Cloud Functions 發 custom claims,列入未來事項。

## 6. 部署

- **網頁**:push `main` → GitHub Pages 自動部署(約 1 分鐘)
- **RTDB 規則**:Console → Realtime Database → 規則 → 貼 `database.rules.json` → 發布
- **Storage 規則**:Console → Storage → Rules → 貼 `storage.rules` → 發布

### Firebase Blaze 開通(照片/文件功能前置,一次性)

1. Firebase Console → 左下「Spark 方案」→ 升級 **Blaze**(綁信用卡,按量計費,免費額度內 $0)
2. Console → 建構 → **Storage** → 開始使用 → 區域選 **`us-central1`**(美國區才有免費額度:5GB 儲存+每日 1GB 下載,家庭用量 $0;選 asia 較快但無免費額度)→ 完成
3. 貼上 `storage.rules` → 發布
4. Google Cloud Console → 帳單 → 預算與快訊 → 建立預算 **$1** 警示(防呆)
5. Cloud Console → API 和服務 → 確認 **Places API** 已啟用(地圖點擊加入用)
