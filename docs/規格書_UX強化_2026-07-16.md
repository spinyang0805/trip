# UX 強化規格書(2026-07-16 第二批)

> 狀態:實作中。對應需求 R8–R11。

## R8 景點卡片:長按拖曳排序(取代 ▲▼)

- 移除卡片上的 ▲▼ 按鈕(保留 ⇄ 跨天選單)
- 統一改用 Pointer Events 長按拖曳,手機/滑鼠同一套:
  - 按住卡片 **400ms** 進入拖曳模式(手機震動 30ms 回饋);400ms 內移動超過 10px 視為捲動,取消拖曳
  - 拖曳中卡片浮起(scale + shadow),目標位置顯示插入線
  - 放開 → 依現行 rekey 機制重排、序號自動重編、路線重算
  - `INPUT/SELECT/BUTTON` 上按下不觸發拖曳
- 移除舊 HTML5 draggable 實作(bindDrag),避免兩套衝突

## R9 地圖路線:每日可開關疊加 + 越洋線修正

- Day 分頁改為**開關(toggle)**:點一下顯示該日路線,再點一下隱藏;可同時顯示多天
- 亮起=顯示中;預設只顯示第一天
- **Bug 修正**:路線計算失敗時的直線 fallback polyline 從未被追蹤,切換天數不會清除 → 越積越多且亂連(跑到菲律賓的線)。改為納入 clearMap 管理;段落直線距離 >300km 直接略過並 toast 提示(通常是定位錯誤)
- 車程資訊面板顯示最後開啟那天的統計

## R10 右側面板:地圖 ⇄ 景點內容切換

- 桌面版右側面板頂部加切換籤:「🗺 地圖」「📖 景點」
- 點景點卡 📖 → 右側面板切到景點內容(iframe 載入 `poi.html?...&embed=1`),不離開行程頁
- embed 模式的 poi.html 隱藏導覽列、縮小邊距
- 手機版(≤768px)維持整頁跳轉
- 面板提供「↗ 開新頁」

## R11 景點明細:名稱查 Google + 星級

- poi.html 名稱輸入框按 **Enter**(或 🔍 鈕)→ Places `findPlaceFromQuery` → `getDetails`
- 取回並儲存:座標、地址、place_id、**rating(星級)**、**user_ratings_total(評論數)**
- 顯示:`⭐ 4.3(1,234 則評論)` 於標題下方,含更新時間
- mini map 跳到新座標
- poi.html 補載 `libraries=places`
